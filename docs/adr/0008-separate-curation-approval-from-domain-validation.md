# Separate curation approval from domain validation

Promotion through the failure-memory pipeline will distinguish curation approval from domain validation. A project maintainer without deep spacecraft domain expertise may approve a draft for curation quality — source traceability, quote accuracy, schema completeness, confidence/caveat labeling, and consistency with repository language — but must not label it technically validated unless a qualified domain reviewer has assessed the failure-class abstraction and CAST/STPA mapping.

This prevents the project from blocking all progress on unavailable domain experts while still being honest about evidence maturity. Records can move from extracted claim to draft case card to curated case card, but academic claims should distinguish curated artifacts from domain-reviewed or accepted artifacts. The validation status must travel with the artifact and be visible in research outputs.
